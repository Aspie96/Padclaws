About Nostr

Nostr (Notes and Other Stuff Transmitted by Relays) is a protocol for decentralized and censorship-resistant social networking.

Nostr servers (relays) act essentially in the same way as traditional centralized platforms, except that they all communicate with users' devices in (roughly) the same way. Relays are fully independent of each other and can be operated by anybody.

Each Nostr user runs a client, which provides a user interface, and allows interaction with any number of chosen relays. Publishing the same post to multiple relays is trivial. When reading, if the client fetches the same post multiple times it only shows it once.

While from the technical point of view of relay operators each relay is essentially analogous to an independent platform, this is mostly transparent to the user. From the point of view of the user, which is the one that matters the most, the whole Nostr ecosystem is just one decentralized censorship-resistant platform.

Each Nostr user has a cryptographic key pair: a public key, which acts as the public identifier of the user, and the corresponding private key, which is to be kept secret. Data is published to relays in the form of events (there are multiple kinds of event, such as the publication of a note), which have a unique identifier, contain the author's public key and are cryptographically signed.

Computing the signature for an event requires knowing the author's private key, but verifying it only requires the public key. The content of an event can't be modified without invalidating its signature. Clients automatically verify signatures and ignore invalid events.

About Padclaws

Padclaws is a free and open source web client for Nostr. Its source code is hosted on GitHub

a>.

Anyone can host an instance of Padclaws for private or public use. Indeed, Padclaws is served to the browser as a static website, which only requires the basic functionalities of any HTTP server. The official hosted public instance of Padclaws has the same name as the software and is located at padclaws.org.

Besides software freedom and easiness of deployment, Padclaws is focused on robustness, privacy (it never talks to any relay or other server unless manually configured by the user), accessibility (trough WCAG guidelines and the Atkinson Hyperlegible font) and security (trough audited cryptographic libraries).
